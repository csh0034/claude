다음 절차에 따라 프로젝트 전체 검증을 수행하고 결과를 리포트한다.

## 절차

1. `./gradlew check` 를 실행한다 (컴파일 + ktlint + 테스트 + koverVerify 포함)
2. 실행 결과를 아래 리포트 형식으로 구조화하여 출력한다
3. 실패 항목이 있으면 각 항목별 수정 가이드를 제공한다

## 리포트 형식

```
## 검증 리포트

| 검증 항목 | 결과 | 비고 |
|-----------|------|------|
| 컴파일 | ✅/❌ | |
| ktlint | ✅/❌ | 위반 파일 수 |
| 단위 테스트 | ✅/❌ | 통과/실패 수 |
| Konsist 아키텍처 | ✅/❌ | 위반 규칙 |
| Kover 커버리지 | ✅/❌ | 현재 커버리지 % |

### 실패 항목 상세 (실패 시에만)

#### [항목명]
- **원인**: 실패 원인 설명
- **위치**: 파일 경로 및 라인
- **수정 가이드**: 구체적 수정 방법
```

## 추가 옵션

- 검증 성공 시 `./gradlew koverHtmlReport`를 실행하여 커버리지 리포트 경로를 안내한다
- ktlint 위반이 있으면 `./gradlew ktlintFormat` 자동 수정 가능 여부를 안내한다
