다음 절차에 따라 코드베이스의 엔트로피를 스캔하고 리포트한다.

## 절차

### 1. 네이밍 패턴 드리프트 감지

`.claude/rules/architecture.md`의 네이밍 컨벤션 표와 실제 코드를 대조한다:

- `*Port` 인터페이스가 `domain` 패키지에 있는지
- `*UseCase` 인터페이스가 `application` 패키지에 있는지
- `*Service` 클래스가 `application` 패키지에 있는지
- `*Adapter` 클래스가 `infra` 패키지에 있는지
- `*JpaEntity` 클래스가 `infra` 패키지에 있는지
- `*Controller` 클래스가 `ui` 패키지에 있는지
- `*Request`/`*Response` 클래스가 `ui` 패키지에 있는지
- `*Command`/`*Query`/`*Result` 클래스가 `application` 패키지에 있는지
- 컨벤션에 맞지 않는 접미사를 가진 클래스 탐지

### 2. 중복 로직 탐지

- 동일하거나 유사한 메서드가 여러 클래스에 존재하는지 검사
- Mapper 변환 로직의 중복 여부 확인
- 유사한 예외 처리 패턴이 반복되는지 검사

### 3. 문서-코드 정합성 검증

다음 문서와 실제 코드/설정의 일치 여부를 검증한다:

- `CLAUDE.md`의 기술 스택 버전 ↔ `build.gradle.kts` 실제 버전
- `.claude/rules/architecture.md`의 패키지 구조 ↔ 실제 패키지 구조
- `.claude/rules/` 내 규칙 ↔ 실제 코드 준수 여부

### 4. 메모리 GC

`.claude/rules/memory-gc.md`의 규칙에 따라 메모리를 정리한다:

- `.claude/rules/` 내 메모리 관련 파일의 정합성 확인
- 90일 이상 미접근 메모리 식별
- 중복 메모리 탐지
- 제거 후보를 사용자에게 제안

|--------|-----------|-------------|------|

| 위치 1 | 위치 2 | 유사도 | 설명 |
